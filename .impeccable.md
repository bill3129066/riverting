# Riverting Design Context — Minimal Editorial System

## Product & Audience

- **Product**: AI Agent Marketplace with per-second streaming payments on X Layer.
- **Core actors**:
  1. Curators publish agents and monetize usage.
  2. Users discover and run agents, paying per-second.
  3. Platform hosts execution and settles proofs/payments.
- **Primary users**: DeFi traders, AI enthusiasts, hackathon judges.
- **User context**: fast decision-making, trust-sensitive financial interactions, high information density.

## Brand Personality

- **Precise**: no ornamental ambiguity; information architecture is explicit.
- **Editorial**: typography-led hierarchy and composition over UI chrome.
- **Trustworthy**: restrained color usage, high contrast, and stable visual rhythm.

## Aesthetic Direction

**Minimal Editorial** (Bloomberg Terminal discipline × Monocle magazine restraint)

- Dense information presented through typographic hierarchy.
- Large structural whitespace between sections.
- Tight local spacing inside related information clusters.
- Minimal decorative UI; emphasis on rhythm, alignment, and legibility.

## Non-Negotiable Design Principles

1. **Typography drives hierarchy, not boxes**.
2. **Whitespace is structural, not decorative**.
3. **Color is functional, not decorative**.
4. **Motion signals state, not decoration**.

## Color System (Exact Tokens)

All neutrals are warm-tinted (no neutral gray ramps).

- `background`: `#11100d` (near-black, warm tint)
- `surface`: `#181613`
- `surface-elevated`: `#211e1a`
- `border-subtle`: `#2f2a24`
- `border-strong`: `#4a4137`
- `text-primary`: `#f4f1ea` (off-white)
- `text-secondary`: `#c9c1b4`
- `text-tertiary`: `#9b9285`
- `accent`: `#b7862f` (editorial deep gold)
- `accent-muted`: `#3b2f1a`
- `accent-foreground`: `#1a150d`
- `success`: `#4f7a4b`
- `warning`: `#b96c2b`
- `error`: `#b24a3f`

### Usage Constraints

- Accent is reserved for active state, key actions, and critical highlights.
- Body copy defaults to `text-primary`; supporting copy uses `text-secondary` then `text-tertiary`.
- Borders default to `border-subtle`; use `border-strong` only for high-separation boundaries.
- Status colors (`success`, `warning`, `error`) are semantic-only, never decorative.

## Typography System (Exact Fonts)

- **Display/Headings**: `Newsreader` (Google Font)
- **Body/UI**: `DM Sans` (Google Font)

### Type Intent

- Headings: editorial, compact, high contrast against dense data.
- Body/UI: neutral, legible, modern sans for transactional clarity.
- Avoid all-caps for long labels; prefer title/sentence case for readability.

## Spacing Scale

Base scale (4px unit):

- `1` = 4px
- `2` = 8px
- `3` = 12px
- `4` = 16px
- `5` = 20px
- `6` = 24px
- `8` = 32px
- `10` = 40px
- `12` = 48px
- `16` = 64px
- `20` = 80px
- `24` = 96px

Rhythm rule:

- Tight spacing **within** a data group.
- Larger spacing **between** groups/sections.
- Use whitespace as an information boundary before adding borders/backgrounds.

## Component Patterns (Replace “Card Grid Everywhere”)

Use these patterns instead of repetitive boxed cards:

1. **Editorial Sections**
   - Section title + optional kicker + thin rule (`border-subtle`), followed by content blocks.
2. **Definition Rows**
   - Label/value rows with strong typographic contrast for metrics and settlement details.
3. **Data Tables with Typographic Emphasis**
   - Dense tabular views; row separators over full card containers.
4. **Inline Panels**
   - Minimal `surface` strips for grouped controls or warnings; avoid deep nesting.
5. **Split-Column Narratives**
   - Left: context/meta. Right: primary actions or live values.
6. **Status Rails**
   - Vertical/horizontal status markers using semantic color only where state changes matter.

Anti-patterns to avoid:

- Uniform card mosaics for unrelated content.
- Card-inside-card nesting.
- Accent used as decorative fill.
- Animated motion without state change.
